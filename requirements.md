# 📄 Vibecoding 需求文档：ResuMagic (简历魔法师)

## 1. 项目愿景与 Vibe (Vision & Vibe)
* **定位：** 一个面向海投求职者的“动态简历匹配与管理工作台”。
* **痛点：** 解决求职者面对不同岗位 JD 时，简历经历不会包装、不会精准匹配、修改耗时费力的痛点。
* **视觉风格（The Vibe）：** **大厂干练风 / 生产力工具范儿**。
    * **配色：** 以飞书、Notion 为灵感。主色调采用深邃的科技蓝（如 `#0052CC`）或干练的墨黑，背景以纯白和极浅灰（`#F4F5F7`）为主。
    * **界面：** 采用直角或微圆角（`4px` - `6px`），线条克制，字体清晰，拒绝花哨的动效，强调“高效、专业、治愈焦虑”。

---

## 2. 核心功能看板 (MVP Features)

### 模块 A：左侧——职位与 JD 管理舱（Workspace）
* **分类列表：** 支持用户创建不同的“目标岗位分类”（例如：产品经理、数据分析师、运营经理）。
* **JD 暂存区：** 在每个分类下，可以新建多个“公司/具体 JD卡片”。点击某个卡片，右侧即刻加载该岗位的匹配工作区。
* **持久化：** 所有分类和粘贴过的 JD 默认保存在前端 `localStorage` 中，用户刷新页面或第二天打开，历史记录依然在。

### 模块 B：中部——魔法匹配输入区（Input Panel）
* **原始经历输入框（大文本框）：** 用户可以毫无心理压力地丢入自己的“流水账”经历、旧简历片段或日常大白话。
* **目标岗位 JD 框（大文本框）：** 自动联动左侧选中的 JD，也支持手动直接粘贴修改。
* **“AI 一键动态匹配” 按钮：** 点击后触发国内大模型 API（如 DeepSeek、智谱清言、Kimi 等），并在按钮上显示干练的 Loading 状态。

### 模块 C：右侧——STAR 闪耀输出区（Output Panel）
* **STAR 格式 Bullet Points：** AI 提炼并润色后的简历描述，以一句话一条的形式呈现（支持一键复制）。
* **三色高亮魔法（核心亮点）：**
    * 🟢 **绿色高亮：** 命中 JD 的关键词（如“数据分析”、“跨部门沟通”）。
    * 🔵 **蓝色高亮：** 强有力的 Action 动词（如“主导”、“搭建”、“优化”）。
    * 🟠 **橙色高亮：** 量化的 Result 结果/数字（如“提升30%”、“节省10h”）。

---

## 3. 技术栈与工程规范 (Tech Stack)
* **框架：** Next.js (App Router) + Tailwind CSS + Shadcn UI（标准的大厂干练组件库）。
* **大模型对接：**
    * 使用**国内主流大模型 API**（如 DeepSeek-V3/R1 或 智谱 GLM-4），配置在环境变量 `.env.local` 中（后端路由转发，确保 API Key 安全）。
    * 采用 **Stream 流式传输**，让生成的文字像打字机一样逐字吐出，提升用户等待时的体验。
* **数据存储：** 纯前端 `localStorage`（MVP 阶段暂不引入复杂数据库，降低部署成本）。

## 4. 核心提示词工程 (System Prompt 内置)
这是决定 AI 输出质量的关键。我们需要在后端请求 API 时，强行注入以下系统提示词：

```text
你是一位拥有10年经验的顶级大厂资深 HR 兼简历修改专家。
你的任务是将用户的[原始经历]，根据[目标岗位JD]进行动态匹配和润色，输出可以直接写进简历的 Bullet Points。

【硬性规则】：
1. 必须严格遵循 STAR 法则。
2. 语言必须大厂化、专业化，严禁假大空，必须结合 JD 中的关键词。
3. 必须包含量化结果（如数字、百分比），如果原始经历中没有，请根据常理推断并用 [数字X%] 或 [具体结果] 留出占位符提醒用户修改。
4. 返回格式必须为 HTML 标签格式，以便前端进行高亮渲染：
   - 命中JD关键词用 <span class="text-green-600 font-medium">关键词</span> 包裹
   - 强动词用 <span class="text-blue-600 font-medium">动词</span> 包裹
   - 量化结果用 <span class="text-orange-600 font-medium">结果数字</span> 包裹

【输出示例】：
- <span class="text-blue-600 font-medium">主导</span>了用户增长项目，利用 <span class="text-green-600 font-medium">Python进行数据分析</span> 洞察用户流失漏斗，<span class="text-blue-600 font-medium">优化</span>了注册留存策略，最终使次日留存率<span class="text-orange-600 font-medium">提升了 15%</span>。

```
